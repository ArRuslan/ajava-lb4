Створити консольний чат.
Програма складається з трьох частин, кожна з яких є консольним додатком.

1. Server.
+ 1.1. Забезпечує прийом з'єднань від клієнтів, їхню аутентифікацію, ретрансляцію повідомлень усім клієнтським додаткам Chat.
+ 1.2. Зберігає список користувачів у налаштуваннях (наприклад, у файлі властивостей users.properties, у форматі login=password).

2. Client.
+ 2.1. Дозволяє клієнту приєднатися до Server і передавши login/password автентифікуватись.
+ 2.2. Програма дозволяє вводити інформацію, яка з'явиться у всіх підключених до сервера (Server) чатах (Chat).

3. Chat.
+ Забезпечує перегляд клієнтських повідомлень. Отримує інформацію від сервера (Server).

4. Приклад роботи всіх програм.

Запустити
1) Server;
2) кілька додатків Chat;
3) кілька програм Client.

Написати пакетний файл Windows (файл командного рядка *nix), який виконує запуск програм.

5. Додаткова інформація.
+ 5.1. Клієнти (Client) можуть надсилати повідомлення серверу (Server), який розсилає їх всім підключеним до нього чатам (Chat).
+ 5.2. Програму Chat завершувати закриттям консольного вікна програми. У додатку Client вихід здійснювати шляхом введення слова exit, при цьому сервер розсилає решті клієнтів повідомлення, що даний клієнт залишив чат.
+ 5.3. Зробити додаток додаток Shutdown, який зупиняє Server. Перед тим, як завершити своє виконання, Server розсилає всі підключені клієнтські програми (Clent і Chat) повідомлення, що сервер припинив роботу.
+ 5.4. Передбачити закриття всіх ресурсів, що відкриваються. Жоден із учасників працюючої програми не повинен отримувати низькорівневі повідомлення про помилки (виключення).
- 5.5. Опціонально: у коді Server-а використовувати логер JUL (Java Util Logging).
